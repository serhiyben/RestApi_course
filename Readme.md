## Що реалізовано в Лабі №4
У цій роботі було здійснено перехід від реляційної бази даних (PostgreSQL) до гнучкої NoSQL бази даних (**MongoDB**) з використанням повністю асинхронного стеку.

### Основні зміни:
1. **Асинхронна NoSQL БД:** Замість синхронного SQLAlchemy використано асинхронний драйвер `motor` для роботи з MongoDB. Це дозволяє API обробляти більше запитів одночасно без блокування потоку.
2. **Гнучка схема даних:** Завдяки інтеграції `Pydantic` та MongoDB, записи зберігаються як JSON-документи з автоматичною генерацією унікальних `ObjectID` (замість класичних числових ID). Це скасовує потребу у складних міграціях таблиць.
3. **Оптимізоване тестування:** Повністю переписані тести під асинхронне середовище за допомогою `pytest-asyncio` та асинхронного клієнта `httpx`.
4. **Ізоляція тестового середовища:** Реалізовано механізм "перевизначення залежностей" (dependency overrides), який створює безпечне підключення до бази під час тестів і вирішує конфлікти з Event Loop.

## Як запустити
1. Встановити залежності: `pip install -r requirements.txt`
2. Запустити сервер: `uvicorn app.main:app --reload`
3. Документація Swagger: `http://127.0.0.1:8000/docs`

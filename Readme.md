
#  Лабораторна робота №9: Навантажувальне тестування API (Advanced Level)

Цей репозиторій містить фінальну версію REST API бібліотеки, яка пройшла комплексне навантажувальне тестування за допомогою **Locust**. Інфраструктура проєкту розгорнута в Docker-контейнерах та включає бази даних, кешування, авторизацію та захист від спаму.

##  Архітектура проєкту
Система побудована за сучасними стандартами мікросервісної архітектури:
* **FastAPI:** Асинхронний веб-фреймворк (мозок програми).
* **MongoDB (через Motor):** Основна NoSQL база даних для зберігання книг (асинхронні операції).
* **Redis:** In-memory сховище, яке використовується для швидкої роботи Rate Limiter'а (алгоритм Sliding Window).
* **JWT + passlib (bcrypt):** Система безпеки та хешування паролів.
* **Docker Compose:** Оркестрація всіх сервісів у єдиній віртуальній мережі.

---

##  Сценарії тестування (Locust)

У межах роботи було реалізовано просунутий сценарій користувацької поведінки (**Complex User Flow**), який імітує реальне навантаження на систему (Read-Heavy workload). 

Віртуальні користувачі виконували дії з різною ймовірністю (вагами):
1. **`on_start` (Логін):** Одноразова авторизація та отримання JWT-токена при старті сесії.
2. **`Weight 10` (Public Read):** Анонімний перегляд загального каталогу книг (`GET /books/public`).
3. **`Weight 5` (Private Read):** Перегляд каталогу з перевіркою JWT-токена (`GET /books/`).
4. **`Weight 1` (Write):** Створення нової книги з авторизацією (`POST /books/`).

##  Результати навантажувального тестування

Тестування проводилося на 30-50 одночасних користувачах (контейнер Locust) з інтенсивним виконанням змішаних операцій.

### Основні метрики:
* **Загальний RPS (Requests per Second):** ~30-50 (обмежено навмисно для імітації реалістичних затримок мережі).
* **Відсоток помилок (Failures):** **0%** (Система успішно обробила всі запити без таймаутів та падінь).

### Аналіз часу відповіді (Response Times):
1. **Швидкодія JWT (Авторизоване читання):** Займає **~4 мс**. Оверхед на розшифровку та валідацію токена становить всього ~1 мс порівняно з публічним читанням (3 мс).
2. **Запис у базу (POST запити):** Займає в середньому **~10-12 мс**, що втричі довше за читання. Це підтверджує очікувану поведінку БД (витрати часу на виділення пам'яті та індексацію).
3. **Вузьке місце (Авторизація):** Логін (`POST /token`) виявився найресурсоємнішою операцією (**~200-500 мс**). Це пояснюється використанням криптографічного алгоритму **bcrypt**, який навмисно споживає процесорний час для захисту системи від Brute-Force атак.

---

##  Висновки

1. **Асинхронність працює:** Зв'язка FastAPI + Motor довела свою ефективність. Навіть під час інтенсивного читання/запису Event Loop не блокується, забезпечуючи час відповіді бази даних у межах 3-10 мс.
2. **Надійність інфраструктури:** Docker-мережа стабільно витримує перехресні запити між FastAPI, MongoDB та Redis без втрати пакетів.

##  Як запустити проєкт та тести

**1. Запуск інфраструктури:**
```bash
docker-compose up -d --build
```
*Доступні сервіси:*
* API (Swagger): `http://localhost:8000/docs`
* Mock API (Prism): `http://localhost:4010`
* Locust Web UI: `http://localhost:8089`



**. Запуск тесту:**
* Перейдіть на `http://localhost:8089`.
* Вкажіть бажану кількість користувачів (наприклад, 50) та швидкість підключення (Spawn rate: 2).
* Натисніть **Start swarming**.

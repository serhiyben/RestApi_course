## Що реалізовано в Лабі №3
У цій роботі було здійснено перехід від традиційної Limit-Offset пагінації до більш продуктивної **Cursor-based пагінації**.

### Основні зміни:
1.  **Оптимізація БД:** Замість відступу записів (`OFFSET`), який сповільнює запити на великих обсягах даних, використано фільтрацію за унікальним ідентифікатором (`ID > cursor`).
2.  **Нова структура відповіді:** API тепер повертає об'єкт, що містить:
    * `items`: список знайдених книг.
    * `next_cursor`: UUID останньої книги на сторінці для завантаження наступної порції даних.
3.  **Стабільність:** Курсорна пагінація запобігає дублюванню або пропуску записів, якщо під час гортання списку було додано або видалено нові книги.

## Як запустити
1. Встановити залежності: `pip install -r requirements.txt`
2. Запустити сервер: `uvicorn app.main:app --reload`
3. Документація Swagger: `http://127.0.0.1:8000/docs`

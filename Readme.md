## Що реалізовано в Лабі №6
У цій лабараторній роботі реалізовано автентифікацію користувачами з використанням JWT токенів. 


### Основні зміни та досягнення (Лабораторна робота №6):
1. **JWT Автентифікація (Access & Refresh Flow):** Реалізовано безпечну Token-based автентифікацію. Додано генерацію короткострокового `access_token` (15 хвилин) та довгострокового `refresh_token` (7 днів) для безперебійного доступу користувачів за допомогою бібліотеки `PyJWT`.
2. **Хешування паролів та керування користувачами:** Створено нову колекцію `users` у MongoDB. Реалізовано ендпоінти реєстрації та логіну. Паролі не зберігаються у відкритому вигляді — використовується надійне криптографічне хешування алгоритмом `bcrypt` (через `passlib`). Додано можливість перегляду та видалення власного профілю (`/users/me`).
3. **Захист маршрутів (Authorization):** За допомогою `FastAPI Depends` та `OAuth2PasswordBearer` налаштовано глобального "охоронця" (`get_current_user`). Усі CRUD-операції з книгами тепер надійно захищені і вимагають передачі дійсного токена у заголовку (`Authorization: Bearer`). Автоматично згенеровано інтерактивну форму авторизації у Swagger UI.
4. **Інтеграційне E2E тестування (Sync TestClient):** Тестове середовище спрощено та стабілізовано. Відмова від нестабільних асинхронних тестів на користь вбудованого синхронного `TestClient`. Написано комплексні сценарії (flows), які перевіряють повний життєвий цикл користувача (від реєстрації до видалення ресурсів) та обробку негативних сценаріїв (неправильні паролі, фейкові токени, невалідні ID), використовуючи єдиний Event Loop через фікстури Pytest.
